2 System Design

Microservices trong Monolith

Mỗi module là một bounded context độc lập với 4-layer chuẩn. Giao tiếp internal qua interface injection, event qua Kafka. Sẵn sàng tách service khi cần.

3 Data Flow

Luồng dữ liệu AI Interview

Pipeline 6 bước từ lúc HR tạo phỏng vấn đến khi nhân viên nhận IDP và BGĐ xem dashboard. Mỗi bước publish event lên Kafka để các service downstream consume bất đồng bộ.

End-to-end Pipeline — KR1 + KR2 + KR3 + KR4 Kafka async · Redis cache · AI scoring
①
HR tạo phỏng vấn
React Admin Portal → POST /api/cbi/sessions → AI Orchestrator gRPC → MongoDB: question_bank
HR chọn role + competency framework. AI Orchestrator nhận prompt, gọi LLM sinh câu hỏi STAR + follow-up. Lưu vào MongoDB question_bank. CBI service trả về session_id.
②
Nhân viên thực hiện phỏng vấn
React UI (chat-style) → WebSocket / REST → Redis: interview_session
NV trả lời từng câu qua giao diện chat. Redis lưu session state (timeout 2h). AI theo dõi real-time, tự động đặt câu hỏi đào sâu nếu câu trả lời chưa đủ S-T-A-R.
③
AI Scoring & lưu kết quả
AI Orchestrator → CBI Service → PostgreSQL: interview_sessions → Kafka: interview.completed
AI chấm điểm từng câu theo rubric STAR (0-100). CBI service tổng hợp ai_score + score_breakdown. Lưu PostgreSQL. Publish event interview.completed lên Kafka.
④
Assessment 360° tổng hợp
Consumer: interview.completed → E360 Service → PostgreSQL: assessment_360
E360 service consume event. Thu thập đủ 4 nguồn: AI(x%) + Manager(y%) + Peer(z%) + Test. Khi đủ → tính final_score = weighted sum. Lưu assessment_360. Publish assessment.done.
⑤
IDP Generation
Consumer: assessment.done → AI Orchestrator → PostgreSQL: idp_plans
IDP service nhận assessment. AI phân tích điểm mạnh/yếu theo từng competency. Match với course catalog (cosine similarity). Sinh recommended_courses có match_score. HR duyệt → active.
⑥
Dashboard & Report
Consumer: idp.updated → Analytics Service → Redis: pre-built reports → React Dashboard (RBAC)
Analytics aggregate theo scope (company/block/branch). Cache vào Redis với TTL 1h. BGĐ xem toàn công ty, Khối xem đơn vị mình, Chi nhánh chỉ xem nhân viên trực thuộc. Talent pool tự động cập nhật.

Migration → Microservice

Thứ tự tách — khi nào táchstrategy
# Bước 1 — Tách sớm nhất (AI cost cao)
ai-orchestrator  → đã có proto sẵn, chỉ deploy riêng

# Bước 2 — Ít dependency nhất
idm              → auth service độc lập
notification     → pure consumer, không có upstream

# Bước 3 — Khi analytics query nặng
analytics        → CQRS: read service riêng

# Quy trình tách 1 module:
1. Viết proto file  api/proto/<module>.proto
2. Thêm gRPC server song song HTTP
3. Tạo  cmd/<module>/main.go  binary mới
4. Caller swap: in-process call → gRPC client
   # interface giữ nguyên → zero change caller
5. Remove khỏi  cmd/api/main.go
