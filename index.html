3 Data Flow

Luồng dữ liệu AI Interview

Pipeline 6 bước từ lúc HR tạo phỏng vấn đến khi nhân viên nhận IDP và BGĐ xem dashboard. Mỗi bước publish event lên Kafka để các service downstream consume bất đồng bộ.

End-to-end Pipeline — KR1 + KR2 + KR3 + KR4 Kafka async · Redis cache · AI scoring
①
HR tạo phỏng vấn
React Admin Portal → POST /api/cbi/sessions → AI Orchestrator gRPC → MongoDB: question_bank
HR chọn role + competency framework. AI Orchestrator nhận prompt, gọi LLM sinh câu hỏi STAR + follow-up. Lưu vào MongoDB question_bank. CBI service trả về session_id.
②
Nhân viên thực hiện phỏng vấn
React UI (chat-style) → WebSocket / REST → Redis: interview_session
NV trả lời từng câu qua giao diện chat. Redis lưu session state (timeout 2h). AI theo dõi real-time, tự động đặt câu hỏi đào sâu nếu câu trả lời chưa đủ S-T-A-R.
③
AI Scoring & lưu kết quả
AI Orchestrator → CBI Service → PostgreSQL: interview_sessions → Kafka: interview.completed
AI chấm điểm từng câu theo rubric STAR (0-100). CBI service tổng hợp ai_score + score_breakdown. Lưu PostgreSQL. Publish event interview.completed lên Kafka.
④
Assessment 360° tổng hợp
Consumer: interview.completed → E360 Service → PostgreSQL: assessment_360
E360 service consume event. Thu thập đủ 4 nguồn: AI(x%) + Manager(y%) + Peer(z%) + Test. Khi đủ → tính final_score = weighted sum. Lưu assessment_360. Publish assessment.done.
⑤
IDP Generation
Consumer: assessment.done → AI Orchestrator → PostgreSQL: idp_plans
IDP service nhận assessment. AI phân tích điểm mạnh/yếu theo từng competency. Match với course catalog (cosine similarity). Sinh recommended_courses có match_score. HR duyệt → active.
⑥
Dashboard & Report
Consumer: idp.updated → Analytics Service → Redis: pre-built reports → React Dashboard (RBAC)
Analytics aggregate theo scope (company/block/branch). Cache vào Redis với TTL 1h. BGĐ xem toàn công ty, Khối xem đơn vị mình, Chi nhánh chỉ xem nhân viên trực thuộc. Talent pool tự động cập nhật.

// Producer config chuẩn ISC — SASL SCRAM-SHA-512
func NewProducer(cfg *config.KafkaConfig) (sarama.SyncProducer, error) {
    c := sarama.NewConfig()
    c.Net.SASL.Enable   = true
    c.Net.SASL.User     = cfg.SaslUsername    // từ Vault
    c.Net.SASL.Password = cfg.SaslPassword    // từ Vault
    c.Net.SASL.Mechanism = sarama.SASLTypeSCRAMSHA512
    c.Net.SASL.SCRAMClientGeneratorFunc = func() sarama.SCRAMClient {
        return &XDGSCRAMClient{HashGeneratorFcn: SHA512}
    }
    c.Net.SecurityProtocol = sarama.SecurityProtocolSaslPlaintext
    c.Producer.RequiredAcks = sarama.WaitForLocal
    c.Producer.Retry.Max    = 3
    c.Producer.Return.Successes = true
    c.ClientID = "ai-hr-platform"  // thêm để SYS monitoring

    return sarama.NewSyncProducer(cfg.Brokers, c)
}

// Async logger — KHÔNG await, tách khỏi business logic
func (k *KafkaLogger) LogAsync(level, msg string, fields map[string]any) {
    topic := k.topicFor(level)  // info/warn/crit
    payload, _ := json.Marshal(map[string]any{
        "service": "ai-hr-platform",
        "env":     k.env,
        "level":   level,
        "type":    fields["type"],  // "error_log" | "response_log"
        "message": msg,
        "fields":  fields,
        "ts":      time.Now().UnixMilli(),
    })
    go k.producer.SendMessage(&sarama.ProducerMessage{  // fire-and-forget
        Topic: topic,
        Value: sarama.ByteEncoder(payload),
    })
}

pkg/config/config.go — đọc Vault file trong podgo
func Load(env string) (*Config, error) {
    v := viper.New()

    // 1. Base config (non-secret)
    v.SetConfigFile("deploy/configs/config.base.yaml")
    v.ReadInConfig()

    // 2. Env override
    v.SetConfigFile(fmt.Sprintf(
        "deploy/configs/config.%s.yaml", env))
    v.MergeInConfig()

    // 3. Vault secrets — đọc từ file được mount vào pod
    vaultFile := fmt.Sprintf(
        "/vault/secrets/configuration.%s.json", env)
    if _, err := os.Stat(vaultFile); err == nil {
        v.SetConfigFile(vaultFile)
        v.SetConfigType("json")
        v.MergeInConfig()
    }

    // 4. ENV variables override tất cả (CI/CD inject)
    v.AutomaticEnv()

    var cfg Config
    return &cfg, v.Unmarshal(&cfg)
}

Redis ISC — Sentinel Master-Slave

internal/platform/cache/redis.go — go-redis Sentinelgo
// Redis ISC — Master-Slave Sentinel failover
// Read: ISP-REDIS:36379 · Write: ISP-REDIS:36380
// Master name: redis_master
// Password: từ Vault /vault/secrets/configuration.[env].json

func NewRedisClient(cfg *config.RedisConfig) *redis.Client {
    return redis.NewFailoverClient(&redis.FailoverOptions{
        MasterName:    "redis_master",
        SentinelAddrs: []string{
            "ISP-REDIS:36379",  // read proxy
            "ISP-REDIS:36380",  // write (master)
        },
        Password: cfg.Password,  // từ Vault
        DB:       cfg.DB,        // SYS cấp DB riêng cho project
        PoolSize: 10,
    })
}

// Bắt buộc prefix mọi key theo chuẩn ISC
// VD: "aihr_session:{id}", "aihr_report:{scope}:{period}"
const KeyPrefix = "aihr_"

func Key(parts ...string) string {
    return KeyPrefix + strings.Join(parts, ":")
}
